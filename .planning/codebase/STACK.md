# Technology Stack

**Analysis Date:** 2026-04-17

## Languages

**Primary:**
- Python 3.10+ - Core framework and all modules (`graphiti_core/`, `server/`, `mcp_server/`)

**Secondary:**
- YAML - Configuration files for MCP server and Docker Compose
- Cypher (query language) - Neo4j graph queries
- BASH - Development scripts and Makefile targets

## Runtime

**Environment:**
- Python 3.10+ (required by all pyproject.toml files)
  - Core library: `>=3.10,<4`
  - Server: `>=3.10`
  - MCP Server: `>=3.10,<4`

**Package Manager:**
- `uv` (UV) - Primary package manager for all three packages
- Lockfile: `uv.lock` (generated by uv)
- Pip-compatible for fallback

## Frameworks

**Core Framework:**
- Pydantic 2.11.5+ - Data validation and serialization (`graphiti_core/`)
  - Used for response models, configuration classes, entity definitions

**Web Framework:**
- FastAPI 0.115.0+ - REST API server (`server/graph_service/`)
  - Async-first HTTP server with automatic OpenAPI/Swagger documentation
- Uvicorn 0.30.6+ - ASGI application server for FastAPI

**LLM/AI Integration:**
- OpenAI SDK 1.91.0+ - Primary LLM provider (bundled with core)
  - Supports OpenAI models (GPT-5, GPT-4.1 families)
  - Async client library
- Anthropic SDK 0.49.0+ - Optional LLM provider (via `[anthropic]` extra)
  - Supports Claude 3.5, 3.7, 4.5 families
- Google GenAI SDK 1.62.0+ - Optional (via `[google-genai]` extra)
  - Gemini 1.5, 2.0, 2.5, 3.0 models
- Groq SDK 0.2.0+ - Optional (via `[groq]` extra)
  - Fast LLM inference provider
- VoyageAI SDK 0.2.3+ - Optional embedding provider (via `[voyageai]` extra)

**MCP (Model Context Protocol):**
- MCP 1.9.4+ - Protocol for AI assistant integrations (`mcp_server/`)

**Database:**
- Neo4j Driver 5.26.0+ - Graph database with async support
  - `bolt://` protocol connections
  - Transaction management and query execution
- FalkorDB 1.1.2+, <2.0.0 - Alternative graph database (optional via `[falkordb]` extra)
  - Redis-based graph engine
  - Async support
- Kuzu 0.11.3+ - Optional alternative graph database (via `[kuzu]` extra)
  - In-process embedded graph database
- AWS Neptune - Optional (via `[neptune]` extra)
  - Uses boto3 and opensearch-py for integration

**Testing:**
- Pytest 8.3.3+ - Test runner and framework
  - Markers: `integration`, other custom marks
- Pytest-asyncio 0.24.0+ - Async test support
- Pytest-xdist 3.6.1+ - Parallel test execution
- Pytest-timeout 2.4.0+ - Test timeout management

**Build & Linting:**
- Ruff 0.7.1+ - Fast Python linter and formatter
  - Configuration: `pyproject.toml` [tool.ruff]
  - Rules: E (pycodestyle), F (Pyflakes), UP (pyupgrade), B (bugbear), SIM, I (isort)
  - Line length: 100 characters
  - Quote style: single quotes
- Pyright 1.1.404+ - Static type checker
  - Core library: `typeCheckingMode = "basic"`
  - Server: `typeCheckingMode = "standard"`
  - MCP Server: `typeCheckingMode = "basic"`

**Utilities & Dependencies:**
- Tenacity 9.0.0+ - Retry logic with exponential backoff
- python-dotenv 1.0.1+ - Environment variable loading
- NumPy 1.0.0+ - Numerical computing
- Pydantic-settings 2.4.0+ - Settings management via Pydantic
- httpx 0.28.1+ - Async HTTP client for API requests
- Sentence-transformers 3.2.1+ - Local embedding models (optional, dev dependency)
- Transformers 4.45.2+ - HuggingFace transformers (dev dependency)
- LangChain 0.2.x - Optional LLM framework integration (dev)
- LangGraph 0.2.15+ - Graph-based LLM workflow (dev)
- IPython/Jupyter 4.2.4+ - Interactive notebooks (dev)

**OpenTelemetry (Optional):**
- opentelemetry-api 1.20.0+ - Tracing API (via `[tracing]` extra)
- opentelemetry-sdk 1.20.0+ - Tracing implementation (via `[tracing]` extra)

**Analytics:**
- PostHog 3.0.0+ - Product analytics (built-in dependency)
  - Telemetry collection (can be disabled via POSTHOG_DISABLED environment variable)

## Configuration

**Environment Variables:**

Core Library:
- `OPENAI_API_KEY` - Required for OpenAI LLM and embeddings
- `ANTHROPIC_API_KEY` - For Anthropic Claude models (optional)
- `GOOGLE_API_KEY` - For Google Gemini models (optional)
- `GROQ_API_KEY` - For Groq LLM provider (optional)
- `VOYAGE_API_KEY` - For Voyage embeddings (optional)
- `USE_PARALLEL_RUNTIME` - Enable Neo4j parallel runtime (boolean, defaults to false)
- `SEMAPHORE_LIMIT` - Max concurrent operations (default: 20)
- `EMBEDDING_DIM` - Embedding dimension size (default: 1024)
- `CHUNK_TOKEN_SIZE` - Token size for text chunks (default: 3000)
- `CHUNK_OVERLAP_TOKENS` - Token overlap for chunks (default: 200)
- `CHUNK_MIN_TOKENS` - Minimum tokens per chunk (default: 1000)
- `CHUNK_DENSITY_THRESHOLD` - Density threshold for chunks (default: 0.15)
- `ENTITY_INDEX_NAME` - Fulltext index name for entities (default: 'entities')
- `EPISODE_INDEX_NAME` - Fulltext index name for episodes (default: 'episodes')
- `COMMUNITY_INDEX_NAME` - Fulltext index name for communities (default: 'communities')
- `ENTITY_EDGE_INDEX_NAME` - Fulltext index name for entity edges (default: 'entity_edges')
- `POSTHOG_DISABLED` - Disable PostHog telemetry (optional)

Server (`graph_service/`):
- `OPENAI_API_KEY` - Required for LLM operations
- `NEO4J_URI` - Database connection URI (e.g., `bolt://localhost:7687`)
- `NEO4J_USER` - Database username
- `NEO4J_PASSWORD` - Database password (note: never commit .env files)

MCP Server:
- `FALKORDB_URI` - FalkorDB connection (e.g., `redis://localhost:6379`)
- `FALKORDB_PASSWORD` - FalkorDB/Redis password (optional)
- `FALKORDB_DATABASE` - FalkorDB database name (default: 'default_db')
- `GRAPHITI_GROUP_ID` - Group partition identifier (default: 'main')
- `SEMAPHORE_LIMIT` - Concurrency limit for operations
- `CONFIG_PATH` - Path to YAML config file
- `BROWSER` - Enable FalkorDB web UI (1=enabled, 0=disabled)

**Configuration Files:**
- `pyproject.toml` - Project metadata, dependencies, tool configurations (root, server/, mcp_server/)
- `Makefile` - Development task automation
- `.env` file - Environment variables (not committed, `.gitignore` excludes)
- `config/config.yaml` - MCP server configuration (YAML)
- `docker-compose.yml` - Container orchestration for server deployment
- `docker/docker-compose-neo4j.yml` - Alternative compose with Neo4j backend

## Platform Requirements

**Development:**
- Python 3.10 or higher
- uv package manager or pip
- Git for version control
- Docker/Docker Compose for containerized development (optional but recommended for MCP server)

**Production:**
- Python 3.10+ runtime environment
- Graph database instance: Neo4j 5.26+ OR FalkorDB 1.1.2+ (mandatory)
- Network connectivity to LLM provider API (OpenAI, Anthropic, Gemini, or Groq)
- Network connectivity to embedding provider (OpenAI or Voyage)
- Optional: Redis for FalkorDB backend

**Deployment Targets:**
- Docker containers (published to Docker Hub: `zepai/graphiti` and `zepai/knowledge-graph-mcp`)
- Kubernetes clusters via container images
- Traditional Python ASGI server deployments (Uvicorn, Gunicorn+Uvicorn)

## Build System

**Build Backend:** hatchling

**Build Configuration Files:**
- `[build-system]` in pyproject.toml specifies hatchling
- Wheel packages built with `hatchling.build`

**Development Commands:** (See `Makefile`)
- `make install` - Install all dependencies with dev extras
- `make format` - Ruff import sorting + formatting
- `make lint` - Ruff checks + Pyright type checking
- `make test` - Run unit tests (integration tests disabled by default via `DISABLE_FALKORDB=1 DISABLE_KUZU=1 DISABLE_NEPTUNE=1`)
- `make check` - Run all: format, lint, test
